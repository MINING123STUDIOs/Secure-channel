🔐 Secure Channel Assistant v2

🧭 How this works (simple view)

1. Key generation
Your browser creates a private key (secret) and a public key (shareable). The private key never leaves your device.

2. Key exchange
When you paste someone else’s public key, both sides combine secrets to create a shared “conversation key”.

3. Encryption
Your message is turned into unreadable ciphertext using AES encryption. Without the shared key, it looks like random noise.

4. Authentication
A built-in integrity check ensures the message was not modified or faked. If tampered, decryption fails.

5. Decryption
Only someone with the matching shared key can restore the original message.

Note: This is a browser-based simulation for learning cryptography concepts. It is not intended for real-world secure communication.
